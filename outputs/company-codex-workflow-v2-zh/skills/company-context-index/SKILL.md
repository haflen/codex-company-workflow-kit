# 公司上下文索引

## 目的

在进入需求、设计、实现、bugfix、spike 或 hotfix 前，建立最小必要项目上下文。

## 工作流

1. 阅读 `specs/global/INDEX.md`。
2. 确认当前产品、版本、里程碑、技术栈、命令和入口文件。
3. 只读取与当前任务相关的 spec、源文件和测试文件。
4. 如果索引过期，记录需要更新的字段。
5. 输出本次任务的上下文摘要和下一步建议。

## 输出

- 项目：
- 当前阶段：
- 相关 spec：
- 相关源码：
- 常用命令：
- 当前不确定性：
- 下一步：

## 约束

- 不要全量扫描文档。
- 不要在上下文阶段编辑实现代码。
- 缺失信息可以标记为待用户或项目负责人补充。
